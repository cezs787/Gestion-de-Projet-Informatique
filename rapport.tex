\subsection{[C3] Gestion de submodule}

Nous avons intégré à notre projet un submodule Git afin de gérer une dépendance externe. 
Pour démontrer l'utilisation des commandes d'intégration, le \texttt{Projet-JS} fera office de 
sous-module dans notre dépôt principal. \\
L'usage de la fonctionnalité submodule peut se montrer pertinente, lorsque le projet en cours dépend 
du contenu d'un autre projets ; son accès et sa visualisation sont alors facilités.

\subsubsection*{Ajout du submodule}

Pour ajouter le submodule au repertoire principal, il faut avoir recours à la commande suivante :

\begin{verbatim}
git submodule add https://github.com/cezs787/Projet-JS extern/Projet-JS
\end{verbatim}

Laquelle a créé automatiquement un fichier \texttt{.gitmodules} contenant la configuration du 
submodule.

    \includegraphics[width=0.5\textwidth]{C3.png}
	\\
    \small Figure Fichier \texttt{.gitmodules} après ajout du sous-module.
\end{center}

\subsubsection*{Utilisation par un nouvel utilisateur}

Pour pouvoir acceder au submodule, il est nécessaire pour le nouvel utilisateur de cloner 
correctement le projet avec le submodule, il doit alors avoir recours à la commande suivante :

\begin{verbatim}
git clone --recursive https://github.com/<nom-du-depot>.git
\end{verbatim}

Si le dépôt a déjà été cloné sans l'option \texttt{--recursive}, l'initialisation se fait via :

\begin{verbatim}
git submodule update --init --recursive
\end{verbatim}

\subsubsection*{Mise à jour du submodule}

Pour synchroniser le submodule avec la dernière version disponible dans son dépôt d'origine :

\begin{verbatim}
git submodule update --remote
\end{verbatim}

\subsection{[C4] Gestion par l'outil Project GitHub}

Pour organiser la gestion et la répatition des différentes taches et étapes du projet, nous avons 
utilisé l’outil \textit{Projects}, plus précisement son tableau Kanban.

Ce tableau est composé de plusieurs colonnes représentant les étapes classiques d’un flux de travail 
:

\begin{itemize}
    \item \textbf{Backlog} : idées et tâches à planifier ;
    \item \textbf{Ready} : tâches prêtes à être commencées ;
    \item \textbf{In progress} : tâches en cours de réalisation ;
    \item \textbf{In review} : tâches en attente de relecture ou de vérification ;
    \item \textbf{Done} : tâches finalisées.
\end{itemize}

Sur le tableau (Figure 1), apparaissent les differentes taches définies par le groupe et leur état de 
progression ; son utilisation nous a permis de nous réferer à des taches précises lors de nos 
discussions sur le projet.\\

Pour assurer l'interactivité du groupe avec le tableau, nous avons accordé les droits 
d'administrateur à chacune des membres du groupe.
